android 6.0 Changes

发表于 2016-05-19   |  

Runtime Permissions

运行时权限

这个版本引入了一个新的权限模型,用户现在可以在运行时直接管理应用程序的权限。这个模型提供了改进的可见性和控制用户权限,同时简化安装和自动升级过程对于应用程序开发人员。用户可以授予或撤销权限单独安装的应用程序。

在你的应用程序的目标Android 6.0(API级别23)或更高,确保在运行时检查和请求权限。来确定应用程序已经被授予权限,调用newcheckSelfPermission()方法。请求权限,调用新的requestPermissions()方法。即使你的应用程序并不是针对Android 6.0(API级别23),你应该测试你的应用在新的权限模型。

  • 有关支持新的权限模型在你的应用程序
    https://developer.android.com/training/permissions/index.html
  • 对如何评估对应用程序的影响,看
    https://developer.android.com/training/permissions/best-practices.html#testing

Apache HTTP Client Removal

移除了联网使用的HttpClient

Android 6.0版本移除了Apache Http Client。如果你的应用程序使用此客户端和目标Android 2.3(API级别9)或更高,使用HttpURLConnection类。这个API是更有效的,因为它通过透明的压缩和响应缓存,减少网络使用和减少能耗。继续使用Apache HTTP api,您必须首先声明以下yourbuild.gradle文件中编译时依赖项:

  android {
    `useLibrary 'org.apache.http.legacy'`
}

Notifications

通知栏方法的删除&修改

这个版本删除Notification.setLatestEventInfo()方法。使用通知。构建器类来构建通知。反复更新通知,重用Notification.Builder实例。调用build()方法来获取更新通知实例。

The adb shell dumpsys notification命令不再打印出你的通知文本。
使用adb shell dumpsys notification --noredact而不是打印的文本通知对象

AudioManager Change

音频调整

设置音量直接或通过AudioManager静音特定流类是不再支持。setStreamSolo()方法已经过时,你应该调用requestAudioFocus()方法。同样,setStreamMute()方法已经过时.已经使用adjustStreamVolume()方法进行取代.
adjustStreamVolume()方法并传递价值ADJUST_MUTE或者ADJUST_UNMUTE的方向。

Text Selection

文本内容编辑修改

当用户选择文本在你的应用程序,现在您可以显示文本选择 比如 Cut,Copy和Parse在Floating ToolBar。用户交互实现类似于上下文操作栏,使所述上下文动作模式为个人观点。

实现浮动工具栏的文本选择,在你现有的应用程序做以下更改:

  1. 在你的View或Activity对象,通过调用startActionMode(回调)或者调用startActionMode(回调,ActionMode.TYPE_FLOATING)改变你的ActionMode调用。
  2. 把你现有的ActionMode.CallBack,替换为使用继承ActionMode.Callback2。
  3. 重写onGetContentRect()方法提供的坐标Rect对象(如文本选择矩形)的观点。
  4. 如果矩形定位不再是有效的,只有一个元素刷新,调用invalidateContentRect()方法。

如果您使用的是Android支持库版本22.2,请注意,floating toolbars不向后兼容和appcompat需要控制ActionMode默认对象。这可以防止浮动工具栏显示。使ActionModesupport AppCompatActivity,调用getDelegate(),然后调用setHandleNativeActionModesEnabled()返回的AppCompatDelegate对象和输入参数设置为false。这个调用返回控制ActionMode对象的框架。在设备运行Android 6.0(API级别23),允许框架支持ActionBar或浮动工具栏模式,在设备上运行Android 5.1(API级别22)或更低,只有ActionBar模式支持。

Android Keystore Changes

签名文件加了重复校验

在这个版本,安卓不再支持DSA密钥库提供者。ECDSA仍支持。

密钥不需要加密静止不再禁用安全锁定屏幕或重置时删除(例如,由用户或设备管理员)。密钥需要加密静止在这些事件将被删除

hexo+github建站

发表于 2016-05-11   |  

准备工作

  1. 初步准备
    安装git 下载地址：git下载
    由于熟悉了linux开发环境,在windows系统中安装了完整版cmder(内置git)
  2. github上创建账号
    • github注册账号
    • 建立repository

      建立仓库名必须是XXX.github.io可以参考wangxujie.github.io

使用hexo

1.安装node.js

下载地址： 下载nodejs
安装之前先验证node和npm是否安装成功(由于新版的NodeJS已经集成了npm)，在cmd中输入node -v npm -v

  1. 安装hexo
    npm install -g hexo这里有warning可以不用管。
    安装hexo

  2. Hexo使用

新建一个文件夹，我在F盘建了D:/hexo
右击打开git bash (或者直接打开cmder)执行
hexo init
文件夹自动生成建网站所需的文件。

npm install
会在文件夹下安装node_modules本地测试

hexo server
然后就可以http://localhost:4000/ 中打开了
记得关server是ctrl+c

创建自己的新博客

  1. 在cd到D:/hexo，
    1. 创建博客
      hexo new "Hello World"
    2. 生成建站资源
      hexo g
    3. 发布到博客
      hexo d
    4. 发布之前可以先在本地发布测试
      hexo s

      看看http://localhost:4000/ 中新发布的blog

编辑文章

在D:\hexo\source_posts中有md文件

可以使用markdown来写博文

自行下载markdown语法的编辑器。（图片外链可以选择图床）

在win10系统需要安装markdown编译环境

部署到Github

  1. 修改_config.yml配置文件
    注意可以放到C++等的一些IDE里修改，我用了C-Free, 不要用文本文件修改 = =。每个：后面需要有一个空格否则会发生错误。

  2. 修改_config.yml最后配置信息 

    deploy:  
   type: git  
   repository: ssh://github.com/wangxujie/wangxujie.github.io.git
   branch: master

  3. deploy的type改成git，然后运行下

    npm install hexo-deployer-git --save
    再

    hexo g
hexo d

有些人将Type改为 github。可能没问题，但我出现了下面的问题Deployer not found: github。

部署成功

可以去你的xxx.github.io看看了~

主题下载

这里注意下你执行命令的文件夹，首先需要在cd themes 中执行clone
更多主题 https://github.com/hexojs/hexo/wiki/Themes

其他

md文件发布问题
在md文件中写中文，发布后会出现乱码的情况，另存为时改为“UTF-8”即可。

优化Hexo

添加“Fork me on Github” ribbon

给blog主页添加一个“Fork me on Github”的绶带（ribbon，比如选择了红色的ribbon，将相应代码复制到Hexo正在使用的theme下layout.ejs中

样式选择 https://github.com/blog/273-github-ribbons

粘贴到 themes\当前使用主题\layout\layout.ejs中，放置在 最后，标签</body>之前即可。
